## Sobre o projeto

**LISBlog** é um projeto pessoal desenvolvido durante o curso de Análise e Desevolvimento de Sistemas na **São Paulo Tech School (SPTech)**.

A ideia nasceu da junção de três grandes paixões: **Life is Strange**, **Tecnologia** e **Escrita**. Assim como Max Caulfield usa seu diário para organizar pensamentos, registrar memórias e entender o peso de suas escolhas, o LISBlog propõe o mesmo exercício: escrever "capítulos" do próprio dia, associar um sentimento a cada um deles e acompanhar, ao longo do tempo, como as emoções e experiências vão se conectando.

Mais do que um blog, o projeto é pensado como uma **ferramenta de autorreflexão e autoconhecimento**, apoiada em conceitos de escrita expressiva (James Pennebaker) e na diferença entre emoção e sentimento trabalhada no eixo socioemocional do curso. Colocar um nome no que se sente ajuda a organizar o caos mental, e é exatamente esse gesto que o LISBlog incentiva.

## Funcionalidades

| Área | Funcionalidade |
|---|---|
| 👤 **Conta** | Cadastro, login e personalização do perfil (nickname e foto) |
| ✍️ **Postagens** | Criação e exclusão de postagens em texto livre, com upload de imagem opcional |
| 🎭 **Emoções** | Seleção de um sentimento por post a partir de uma lista fixa (Alegria, Alívio, Ansiedade, Culpa, Esperança, Medo, Paz, Raiva, Tristeza) |
| ⭐ **Memórias especiais** | Marcação de posts como "memória importante", reunidos em um painel estilo polaroid, assim como a parede de fotos do jogo |
| 🧵 **Feed social** | Visualização das postagens de todos os usuários e curtidas |
| 📊 **Dashboard** | Total de capítulos, emoção mais frequente, curtidas/comentários recebidos e gráficos (feitos com Chart.js) da distribuição de emoções e da evolução de postagens ao longo do tempo |

## Tecnologias utilizadas

**Front-end**
- HTML & CSS — estrutura e estética inspirada visualmente no jogo
- JavaScript — lógica de interface e feedback de ações/erros
- Chart.js — gráficos da dashboard

**Back-end**
- Node.js  — API responsável pelo sistema de posts, login e interações
- MySQL — banco de dados

**Gestão do projeto**
- GitHub — versionamento de código
- Trello — organização de tarefas e backlog


## Justificativa

Com esse projeto, a ideia é ajudar as pessoas a entenderem que suas ações não devem ser um peso, mas sim parte do que constrói quem elas são hoje. Escrever sobre um acontecimento e conseguir nomear o sentimento predominante é um passo essencial para ter uma visão mais clara sobre si mesmo e sobre a forma como lidamos com nossas próprias escolhas.
